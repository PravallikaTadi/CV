Being an ETL developer and on the source end of the project cycle, the first step is to extract the data from the mainframe files. The second step is to send this raw data into the staging area/landing zone where the data is transformed as per the business logic using SCD Type1/ Type2 processing. Here, the data is cleansed, structured, and tested before loading into the target system. The final step is to load this transformed/processed data into the target i.e., the Oracle database. In this final phase of the ETL process, the data is stored in the form of tables which are further populated into Fact tables and Dimension tables using Star Schema. The downstream uses this data for data visualization and generating reports.
